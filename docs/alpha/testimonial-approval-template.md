# Testimonial Approval Template

Use this only after an alpha tester has explicitly offered public feedback.

Do not publish testimonials, screenshots, company names, handles, or revenue-related claims until the tester approves the exact public wording.

## Approval Request

Thanks for the feedback on 10kmrr.life.

I would like to use the quote below publicly:

> [Exact proposed quote]

Planned attribution:

- Name / handle:
- Company or project, if any:
- Link, if any:

Please confirm:

- [ ] The quote wording is approved exactly as written.
- [ ] The attribution is approved exactly as written.
- [ ] The quote does not reveal exact private MRR.
- [ ] The quote does not reveal Stripe keys, customer/payment data, raw logs, raw Stripe responses, or private screenshots.
- [ ] Any screenshot attached to the quote uses mock, blurred, or intentionally sanitized MRR.

Reply with one of:

- `approved_as_written`
- `approved_with_edits`
- `do_not_publish`

## Notes Boundary

Record only the approval status, approved public wording, attribution, and date.

Do not record:

- Stripe keys.
- Exact private MRR.
- Raw revenue logs.
- Raw Stripe responses.
- Customer names, emails, invoices, payments, subscriptions, or payment methods.
- Unsanitized screenshots.

## Publication Rule

If approval is ambiguous, treat it as `do_not_publish`.
