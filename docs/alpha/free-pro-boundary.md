# Free vs Pro Boundary

10kmrr.life should stay useful as a free, source-available local Mac tool. Pro should pay for convenience, polish, and maintenance, not for holding a user's Stripe data hostage.

## Free Alpha Core

Free alpha includes:

- Build from source.
- Local setup window for restricted Stripe key.
- Keychain storage for the Stripe key.
- Local Stripe subscription fetch.
- Stripe-like MRR calculation.
- Lock Screen overlay.
- Last-good local cache.
- Refresh interval and vertical position settings.
- Safe local diagnostics.

Free alpha must not require:

- A 10kmrr.life account.
- A hosted backend.
- Uploading Stripe keys.
- Uploading revenue data.
- Sharing raw Stripe responses.

## Pro Candidates

Good Pro candidates are convenience or maintenance upgrades:

- Signed and notarized installer.
- Auto-update.
- Compatibility maintenance for macOS releases.
- Additional visual styles.
- More polished setup and troubleshooting UI.
- Size, position, and multi-display controls.
- MRR goal or milestone display.
- Priority support for local install issues.

## Non-Goals

Avoid turning Pro into:

- Full analytics dashboard.
- Churn or cohort reporting.
- Forecasting.
- Team finance workspace.
- Hosted Stripe data warehouse.
- Anything requiring 10kmrr.life to store Stripe API keys.

## Pricing Hypotheses

Test pricing as a signal, not a promise:

- One-time founder license: USD 19 to 49.
- Annual Pro: USD 29 to 79 per year.
- Monthly Pro: USD 5 to 9 per month.

Do not publish final pricing until alpha retention, support load, and compatibility maintenance cost are understood.

## Upgrade Signal

Strong Pro signal exists when a tester:

- Keeps using the free alpha for 7 days.
- Names a specific convenience feature they would pay for.
- Accepts the private macOS API compatibility caveat.
- Chooses a plausible price point.

Weak Pro signal:

- They only want full analytics.
- They would not run a local Mac tool.
- They need a hosted dashboard.
- They object to restricted-key setup.
